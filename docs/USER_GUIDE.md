# FitCamp 打卡 — 用户说明

## 产品简介

FitCamp 是一个**多期打卡挑战**应用：每期需缴纳 **100 USDC** 作为打卡定金，在**规定天数内完成 7 次打卡**的用户视为「获胜者」，可**平分**当期未完成者的定金；未完成 7 次打卡的用户定金将进入奖池。

- **每期时长**：默认 7 天（由群主开启新一期时设定）。
- **结算**：当期结束后，完成 7 次打卡的用户可「提现」；群主在所有人提完后可「提取余数」，再「开启新一期」。

---

## 角色说明

| 角色 | 说明 |
|------|------|
| **群主（K）** | 负责开始/结束打卡、无获胜者时结算、提取余数、开启新一期。打卡由群主在链上确认（用户点「打卡」时实际由群主账户执行）。 |
| **用户（A / B / C）** | 缴纳定金参加当期、每日打卡、期结束后若完成 7 次打卡可提现。 |

页面上方 **「当前用户」** 下拉框可切换身份，切换后会显示对应操作按钮和状态。

## 操作流程

### 1. 群主：开始打卡

- 选择 **当前用户：K（群主）**。
- 点击 **「开始打卡」**。  
  此后用户才可以「缴纳打卡定金」和「打卡」（本演示中「开始」为前端状态，实际链上在群主开启新一期后即可参加）。

### 2. 用户：参加与打卡

- 选择 **当前用户：A / B / C**。
- 点击 **「缴纳打卡定金」**：从当前账户转 100 USDC 到合约，参加**当前期**。
- 点击 **「打卡」**：由群主（K）在链上确认一次打卡。每期需完成 **7 次** 才算获胜。  
  - 本期**已结束**后无法再缴纳定金；若提示「本期已结束」，需等群主「开启新一期」后在新期参加。

### 3. 群主：结束打卡（演示用）

- 选择 **当前用户：K（群主）**。
- 点击 **「结束打卡」**。  
  本演示会推进本地链时间，使当期结束，用户将无法再打卡。

### 4. 用户：提现（获胜者）

- 选择 **当前用户：A / B / C**。
- 若当前账户在**某一期**已参加、完成 7 次打卡且未提现，**「提现」** 按钮会可用。
- 点击 **「提现」**：对**该期**执行结算（若尚未结算）并领取自己那份奖励（平分当期奖池）。

### 5. 群主：提取余数

- 条件：**当期已结算** + **所有获胜者都已提现** + **合约内还有余额**（除法余数等）。
- 选择 **当前用户：K（群主）**，点击 **「提取余数」**，将合约剩余 USDC 提至群主地址。

### 6. 无人完成 7 次打卡时

- 若当期**没有人**完成 7 次打卡，群主需先点击 **「无获胜者结算」**，将当期标记为已结算。
- 然后点击 **「提取余数」** 提走池内资金。
- 再点击 **「开启新一期」**。

### 7. 群主：开启新一期

- 条件：**当前期已结束** + **当前期已结算** + **所有获胜者已提现** + **合约余额为 0**（即已「提取余数」）。
- 点击 **「开启新一期」**，设定下一期天数（默认 7 天），开启新一轮挑战。

## 页面状态说明

**当前状态** 区域会显示例如：

- **当期**：第几期  
- **地址**：当前选中账户地址  
- **ETH / USDC**：该账户余额  
- **已参加当期**：是否已缴纳本期定金  
- **当期打卡次数**：本期的打卡次数（需 ≥ 7 才能提现）  
- **当期已提现**：本期是否已领取奖励  
- **当期结束时间戳**：本期结束的链上时间  
- **当期已结算**：本期是否已结算  

根据上述信息可判断当前能否「缴纳定金」「打卡」「提现」或群主能否「提取余数」「开启新一期」。

## 常见提示与处理

| 提示或现象 | 说明与处理 |
|------------|------------|
| 请先运行: npx hardhat node… | 本地环境未就绪，需由部署者先启动本地链并执行 init-local、再启动前端。 |
| 合约地址与当前链不一致 | 链或合约重新部署过，需重新执行 init-local 并刷新页面。 |
| 本期已结束，无法参加 | 当前期已过期，等群主「开启新一期」后在新期「缴纳打卡定金」。 |
| Round ended | 同上，本期已结束。 |
| 请先让完成 7 次打卡的用户点击「提现」… | 提取余数前需先完成结算；有获胜者时由他们先「提现」触发结算。 |
| 尚有获胜者未提现 | 等所有获胜者都提现后再「提取余数」。 |
| 本期有完成 7 次打卡的用户… | 不能点「无获胜者结算」，应让这些用户先「提现」。 |
| 请先「提取余数」清空合约余额后再开启新一期 | 开启新一期前必须把合约里剩余 USDC 提走。 |

## 技术说明（可选阅读）

- 本页使用**本地开发链**，通过同一站点代理 RPC，不依赖 MetaMask 等钱包插件；若浏览器提示钱包相关冲突，可忽略。
- 用户 A/B/C 及群主 K 的身份由页面内嵌的测试私钥区分，仅用于本地演示，请勿在生产或主网使用。
